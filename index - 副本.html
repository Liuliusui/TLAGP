TLAGP: Tutorial-Learning-Assisted Genetic Programming

A semantically guided GP framework where LLMs act as tutors—not generators. TLAGP enhances crossover, mutation, and fitness shaping using zero-shot subtree scoring to evolve interpretable and reusable policies for CTTS, YCS, and CRP.

TLAGP at a glance

  • Typed GP + LLM semantic scoring, without generating code.
  • Improved operator decisions through subtree importance scores.
  • Cross-problem portability for CTTS, YCS, CRP.
  • Interpretable expression trees with optional LLM rationales.

What & Why

Pain points in current scheduling

  • Hand-crafted rules require deep domain expertise.
  • GP frameworks often must be redesigned per problem (CTTS/YCS/CRP).
  • Crossover/mutation operate blindly and cause code bloat.
  • Lack of interpretability and auditability in automated heuristics.

What TLAGP provides

  • LLMs score subtrees to guide GP operators.
  • A unified typed GP representation reused across tasks.
  • Interpretability via symbolic expression trees + optional rationales.
  • Zero-shot LLM scoring—no training or labels needed.

Key Features

Unified typed GP

A single, reusable GP grammar and primitive set across CTTS, YCS, CRP.

LLM-assisted operator control

Zero-shot subtree scoring guides crossover, mutation protection, and fitness shaping—LLMs serve as tutors, not code generators.

Interpretable & auditable

Symbolic GP trees remain fully readable, with optional LLM rationales supporting audits, risk control, and training.

Cross-problem reuse

Switch tasks by rewriting the domain description only; operators and representation remain unchanged.

Educational/tutorial value

TLAGP logs subtree rationales that help students and practitioners understand why a rule works.

Educational use: learning GP & heuristics

GP made concrete

TLAGP exposes priority functions as expression trees instead of black boxes. Students can examine how features combine into heuristics, making GP easier to teach and learn.

Tutorial guidance from LLMs

Zero-shot subtree scores and rationales highlight which structural components matter and why—supporting classroom demonstration and practitioner training.

Why TLAGP Matters for Project Management

  • Reduced engineering cost: GP representation reused across CTTS/YCS/CRP.
  • Transparency & auditability: Symbolic rules + LLM rationales form traceable logic.
  • Capability building: Tutorial outputs support team onboarding and knowledge transfer.
  • Lower operational risk: Semantic operator control reduces brittle heuristics.
  • Scalable & maintainable: Domain descriptions can be updated without modifying GP internals.

How TLAGP Works

  1. Extract top-k deepest subtrees.
  2. Score subtrees using a modular zero-shot LLM prompt.
  3. Bias crossover, mutation, and fitness shaping using these scores.
  4. Run simulation feedback inside a standard GP loop.

Get Started with TLAGP 

from tlagp import quick_start

def cost_fn(rule):
    return simulate(rule)

result = quick_start(
    prompt="Container relocation. Minimize relocations.",
    cost_fn=cost_fn,
)

print(result.hof[0])

LLM scoring is optional—without credentials, TLAGP falls back to standard GP.
